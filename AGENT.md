# Agent

这个文件定义执行方式。目标不是生成很多内容，而是持续把这套资料打磨成可复习、可包装、可面试的工程资产。

## 工作边界

- 默认作用域：`cpp_interview/`
- 默认目录：`~/cx_ws/cpp_interview`
- 默认对象：笔记优化、项目包装、简历表达、面试准备、学习路径

## 默认工作流

1. 先判断任务属于：知识复习、项目包装、简历优化、面试模拟、流程整理。
2. 再判断当前阶段属于：框架搭建、主线强化、项目打磨、面试冲刺。
3. 最后只给出当前最有效的一步，避免信息过载。

## 执行原则

- 优先基于现有材料组织知识，不脱离上下文空讲概念。
- 优先采用简单、标准、可维护的方案。
- 优先解决结构问题、边界问题和表达问题。
- 每次输出都尽量落到工程场景、项目模块或面试表达。
- AI 负责补框架、补视角、补检查清单；技术判断仍以人的理解为准。
- 对 `导师增强式复习`，允许主动补充必要知识点，不受“原笔记不新增知识”限制。
- 补充内容优先覆盖：经典资料要点、系统调用链路、源码切面、真实工作故障、面试官追问。

## 文档维护规则

- 同一规则或说明尽量只保留一个权威入口，避免跨文件重复定义。
- 非必要不新增实体：不随意新增顶层文档、目录、命名方案或同类流程文件。
- 文件命名稳定优先；如果现有名称已经清楚，就优先保留。
- 优化文档时优先删重复、收边界、统一口径，而不是继续加内容。

## 输出格式

- 知识复习：主线、底层原理、经典补充、工程问题、面试题、项目落点。
- 项目包装：背景、目标、架构、模块、难点、优化、追问。
- 面试模拟：提问、点评、修正、继续追问。
- 流程整理：边界、步骤、验收标准。

## Git 要求

- Git 默认在项目根目录执行。
- 一个明确批次对应一次提交。
- 提交信息必须描述真实改动。
- 具体命令和模板以 [docs/git_workflow.md](docs/git_workflow.md) 为准。
