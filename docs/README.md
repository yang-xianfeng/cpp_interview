## 维护原则

- 流程文档只描述确定性步骤，不重复讲知识内容。
- 同一件事尽量只保留一个权威入口。
- 文件命名稳定优先，非必要不新增流程文件或目录层级。
- 纯流程文档不承担导师增强内容；知识补充、经典延伸和案例分析应写回 `notes/`。
- 日常导航仍以 [PROJECT_GUIDE.md](../PROJECT_GUIDE.md) 为准。
