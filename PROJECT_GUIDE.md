# Project Guide

这个文件只回答三件事：先看什么、在哪里工作、按什么顺序推进。

## 先看什么

1. [README.md](README.md)：确认项目边界。
2. [ROLE.md](ROLE.md)：确认工作视角和目标。
3. [AGENT.md](AGENT.md)：确认执行方式。
4. [CANDIDATE_PROFILE.md](CANDIDATE_PROFILE.md)：确认候选人背景与求职边界。
5. [notes/README.md](notes/README.md)：进入复习主线。
6. [docs/README.md](docs/README.md)：进入流程文档。

## 在哪里工作

- 工作空间：`~/cx_ws`
- 项目根目录：`~/cx_ws/cpp_interview`
- 默认执行位置：`~/cx_ws/cpp_interview`

## 怎么推进

- 知识复习：从 `notes/cleaned/00-杂项.md` 开始，按编号顺序推进。
- 复习方式：不是只做排版整理，而是结合经典资料、源码、面试追问和工作案例做导师增强。
- 项目包装：重点关注 `23-搜索引擎项目.md` 与 `26-Workflow重写NetDisk.md`。
- 面试准备：重点关注 `30-面试经验.md`、`31-面试简历.md`、`32-15天复习计划.md`。
- 流程规范：统一从 `docs/` 进入，不在多个入口重复维护。

## 维护规则

- 入口顺序变化时，优先更新这里，不在多个文件重复解释同一导航逻辑。
- 文件命名稳定优先；非必要不改文件名，不新增同类入口文件。
