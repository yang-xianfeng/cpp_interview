### 4.1 格式化输出：`printf( )`
int printf(格式化字符串，表达式1，表达式2，... )
显示格式化字符串，并在指定位置插入要替换的值
**返回值 ： 显示字符的个数**

格式：%m.p X
X —— 以何种方式解释内存空间
m（minimum field width 最小字段宽度) —— 表明要显示的最少字符数量
p （presion) 精度
精度的含义 依赖 转移说明符 。（ X )
%d 中的 p 代表待显示数字的最小个数 （必要时会在前面补0）。若省略p，则默认为1.

- %5d

=5位 正好

- %-md  	右对齐
- %+md 	左对齐
- %04d       用0填充前面的空格

%d 中的 p 代表小数点后的数字的个数 （默认为6位小数）。如果 p=0，则不显示小数点.

### 4.2 scanf()
int scanf(格式串，表达式1，表达式2，...)
`scanf( )` 本质上是一个“模式匹配函数”，试图把输入的字符与转换说明进行匹配，
转化成对应的二进制，写入到对应的内存空间中。

从左到右依次处理转换说明，如果成功，则继续处理后面的字符串；
如果失败，则立即返回，返回值表示处理转换说明 成功的个数。
**返回值 ：匹配成功的值的个数**

#转换说明：1. 表示匹配的规则；
2.表示将字符转化成对应的二进制数据。
格式串：普通字符：精确匹配
其他字符：精确匹配
空白字符：`**\space**``**\t**``**\n**``**\v**`**  ** 匹配任意多个空白字符，包括**零个**

注意：`scanf( )` 匹配到 %d , %f （进行**数值匹配**时），会**匹配**前面的空白字符。

### 14.2 汉诺塔 (hanoi)
```c
void hanoi(int n, char start, char middle, char target) {
	// base case
	if (n == 1) {
		printf("%c --> %c\n", start, target);
		return;
	}
	// 递推公式:

    //把(n-1)个盘子从start移动到middle上
	hanoi(n - 1, start, target, middle);
	// 把最大的盘子从start移动到target上
	printf("%c --> %c\n", start, target);
    //把(n-1)个盘子从middle移动到target上
	hanoi(n - 1, middle, start, target);
}
```
移动次数：
S(n) =1 		       n=1
 = 2S(n-1) +1  n≥2
=> S(n) = 2n - 1

### 递归的总结：

1. 什么可以考虑使用递归？

       一个大问题可以分解为若干个子问题，且子问题的求解方式与大问题一致。（递）
       可以将这些问题的解合并成大问题的解（归）

2. 使用递归需要注意的问题：

边界条件，避免stackoverflow问题
重复计算

3. 如何写递归：边界条件 +  递推公式

		{3, "zhaolinger", 100, 99, 100}, {4, "xiaolongnv", 100, 100, 99}, {5, "baixiuzhu", 100, 100, 99}};

// 比较规则: 总分从高到低, 语文成绩(高-->低), 数学成绩(高-->低), 英语成绩(高-->低), 姓名(字典顺序从小到大进行比较)
int compare(const void* p1, const void* p2) {
	Student* s1 = (Student*)p1;
	Student* s2 = (Student*)p2;
	int total1 = s1->chinese + s1->english + s1->math;
	int total2 = s2->chinese + s2->english + s2->math;
	if (total1 != total2) {
		return total2 - total1;//从大到小
        // total1 > total2
        // total1 < total2
	}
	if (s1->chinese != s2->chinese) {
		return s2->chinese - s1->chinese;//从大到小
	}
	if (s1->math != s2->math) {
		return s2->math - s1->math;//从大到小
	}
	if (s1->english != s2->english) {
		return s2->english - s1->english;//从大到小
	}
	return strcmp(s1->name, s2->name);
}
