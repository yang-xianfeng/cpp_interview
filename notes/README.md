# Notes Index

默认优先阅读 `notes/cleaned/`；只有在需要回查原始表达或确认清洗前内容时，再对照 `notes/source/`。

## 阅读方式

- `按编号顺序`：`00 -> 01 -> 02 -> ... -> 32`
- `按主题主线`：语言 -> Linux -> 网络 -> 数据库 -> 项目 -> 面试

如果当前目标是重新找工作，建议先按编号顺序推进：

1. 先读 [00-杂项.md](cleaned/00-杂项.md)，建立整体框架。
2. 再按编号顺序往后读，避免知识断层。
3. 每完成一个主题，再回到项目与面试文档做表达整合。

## 单篇复习动作

每篇笔记默认做 5 件事：

1. 写出主线框架。
2. 补底层原理，重点看系统调用、内存、并发、网络链路。
3. 记录工作中常见问题、线上故障点和排查路径。
4. 整理 5 到 10 个高频面试题，并练习口述回答。
5. 关联到项目表达：这一篇知识在真实项目里体现在哪些模块。

在这个项目里，复习不是单纯回看旧笔记，而是 `导师增强式复习`。因此每篇笔记都允许继续补充：

- `Unix 环境高级编程` 及同等级经典资料中的关键结论
- 必要的源码切面或系统实现细节
- 真实工作环境中可能遇到的问题和排查方式
- 面试官一层层追问到底层时的回答路径

复习目标不是“看完”，而是：

- 能讲清原理
- 能落到工程实践
- 能顶住一层到两层追问
