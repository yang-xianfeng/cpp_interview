客观世界：对象之间的交互。

面向对象的**精髓**： **消息传递**。

程序世界： 类 -> 对象   （ 数据成员 、成员函数 ）

C++ 类访问修饰符:

关键字 **public、private、protected** 称为访问修饰符。

一个类可以有**多个** public、protected 或 private 标记区域。

```c
class 类名{
    public:
    // 在类外可以直接访问
    void func();// 成员函数
    protected:
    // 交给子(派生)类时的保护权限
    private:
    // 在类之外无法直接访问，可以通过类中成员函数访问
    int _money; // 数据成员
    char _id[19];
}; // 千万不要忘了这个分号
```

类中的**数据成员** 都要**加 _**  ( _num , _name)

class**默认**访问权限 **private**（ 在C语言中 struct **访问权限 ** **public** ）

C++中struct的功能**被拓展了**，使用与class相同，但是struct默认不设权限时 为public ， class 默认不设权限时 为private 。

不需要设置访问权限可以直接使用struct

示例：

```c
#include<iostream>
#include<string.h>
// 类大括号内部 是 类内，类大括号外部 是 类外
class Computer {   
    // 代码风格 : public在前
    // public下 对外的接口
   public:
    //成员函数
    void setBrand(const char* brand) { strcpy(_brand, brand);}
    void setPrice(float price) { _price = price; }

   private:
    //数据成员  类中的数据成员都要以_命名 ( _num , _name)
    char _brand[20];
    float _price;
};
int main(int argc, char* argv[]) {
    Computer pc; // 实例化对象
    // . 成员访问运算符
    pc.setBrand("Huawei Matebook");
    pc.setPrice(8848);
    return 0;
}
```

#### 空类

一个**空类**占据的空间有多大？会自动创建哪些函数呢？

在C++中空类会占一个字节，为了让对象的实例能够相互区别。

编译器会生成4个成员函数：

- 一个缺省的构造函数
- 一个拷贝构造函数
- 一个析构函数
- 一个赋值运算符

```c
#include <iostream>

using std::endl;

//一个空类会提供哪些函数?

int main(void) {
    Empty e1;       //调用默认(无参)构造函数
    Empty e2 = e1;  //调用拷贝构造函数
    //对象被销毁时，会调用析构函数

    Empty e3;
    e3 = e1;  //调用赋值运算符函数

    return 0;
}
```

### 2.3 析构函数 用来清理 类对象成员函数中申请的资源等

构造函数在创建对象时被系统自动调用

析构函数(Destructor)在对象**超出其作用域**被撤销时被自动调用

析构函数有如下特点：

- 与类同名，之前冠以波浪号，以区别于构造函数。
- 析构函数没有返回类型，也不能指定参数。且 析构函数只能有一个，不能被重载。

析构函数在哪些时候会被调用呢？

1. 全局对象 ： 
   - 当程序开始运行，在主函数**main**接受程序控制权**之前**，就调用**构造**函数创建全局对象
   - **整个程序结束**时，自动调用全局对象的**析构**函数。
2. 关键字static定义的全局静态变量 ： 
   - 当程序开始运行，在主函数**main**接受程序控制权**之前**，就调用**构造**函数创建全局对象
   - **整个程序结束**时，自动调用全局对象的**析构**函数。
3. 关键字static定义的局部静态变量 : 
   - 当程序流程第一次到达该对象定义处调用构造函数
   - 在**整个程序结束时**调用**析构**函数。
4. 局部定义的对象 : 
   - 每当程序流程到达该对象的定义处就调用构造函数
   - 在程序离开**局部对象的作用域**时调用对象的**析构**函数。
5. new运算符创建的对象 : 
   - 每当创建该对象时调用构造函数
   - 用**delete**删除该对象时，调用**析构**函数。

### 2.4 本类中对象的复制（拷贝）

1. 对象未创建 ： 调用 拷贝构造函数

```c
类名 ( const 类名 & rhs )
```

2. 对象已创建，通过赋值语句  `**=**`  直接复制（拷贝）对象

```c
int * pArr =  new int[10]();  // array[]  ()初始化0
```

```c
class Computer{
    public:
    Computer(const char* brand, float price)
        // new表达式申请数组空间时，最后不加() 不会进行初始化
        //: _brand(new char[strlen(brand) + 1])
        // new表达式申请数组空间时，加上() 就会进行初始化
        : _brand(new char[strlen(brand) + 1]()),
    	_price(price) {
        cout << "Computer(const char*, float)" << endl;
        strcpy(_brand, brand);
    }

    Computer(const Computer& rhs)
        : _brand(rhs._brand),  //传递地址本身
        _price(rhs._price) {
        cout << "Computer(const Computer&)" << endl;
    }

    void print() {
    if (_brand) {
        cout << "brand:" << _brand << endl << "price:" 
             << _price << endl;
   		 }
	}
    
    void release() {
        if (_brand) {
            delete[] _brand;
            // nullptr是C++11的写法，更安全
            _brand = nullptr;  //置空
        }
    }

    //析构函数回收的是数据成员申请的空间
    ~Computer() {
        release();
        cout << "~Computer()" << endl;
    }
    
    private:
    // 16B // 64位系统默认情况下按8字节对齐
    char* _brand;
    float _price;
}
```

析构函数的调用 和 本对象销毁 是一回事吗？

数据成员的资源回收 需要调用 析构函数

对象销毁时一定调用析构函数，但调用析构函数时不一定 销毁对象

虽然析构函数可以显式调用，但应该让它**自动调用**。

注意：

当使用**cout**输出**char _的nullptr_ ， 会导致**程序崩溃退出**

### 2.7赋值运算符 重载 （面试高频考点 ）

系统自动提供一个赋值运算符

```c
className & operator=(const className &rhs); // 双目运算符(this)
```

```c
className & operator+(const className &lhs,const className &rhs );
```

**赋值运算符重载 （ 深拷贝 ）    必知必会**

```c
Computer &Computer::operator=(const Computer &rhs) {
        if (this != &rhs)  // 1、考虑 自复制
        {
            delete[] _brand;  // 2、释放左操作数对象申请的空间
            _brand = nullptr;
            _brand = new char[strlen(rhs._brand) + 1]();  // 3、深拷贝
            strcpy(_brand, rhs._brand);
            _price = rhs._price;
        }
        return *this;  // 4、返回*this
    }
```

步骤：

1. 考虑 自复制
1. 释放左操作数对象原来申请的空间
1. 进行**深拷贝**
1. **返回_this_

```c
Computer &Computer::operator=(const Computer &rhs){
    if(this != &rhs) //1、自复制
    {
        delete [] _brand; //2、释放左操作数
        _brand = nullptr;
        
        _brand = new char[strlen(rhs._brand) + 1](); //3、深拷贝
        strcpy(_brand, rhs._brand);
        _price = rhs._price;
    }
    
    return *this; //4、返回*this
}
```

### 指向对象的指针

对象占据一定的内存空间，和普通变量一致， C++ 程序中采用如下形式声明指向对象的指针：

```c
类名 *指针名 [=初始化表达式];
```

指针中存储的是**对象**所占内存空间的**首地址**。

初始化表达式是可选的:

- 通过取地址（&对象名）给指针初始化，
- 通过申请动态内存给指 针初始化，
- 干脆不初始化（比如置为nullptr），在程序中再对该指针赋值。

```c
Point pt(1, 2);
Point *p1 = nullptr;
Point *p2 = &pt;
Point *p3 = new Point(3, 4);
Point *p4 = new Point[5];
p3->print();//合法
(*p3).print();//合法
```

## 单列模式（ 类 --实例化--> 对象）  （面试高频考点 ）

一个类只能生成一个对象，且是唯一的对象

实现步骤：

1. 将构造函数私有化 （ 该对象不能是一个 栈对象、全局/静态对象，只能是一个 堆对象 ）
1. 在类中定义一个静态的指向本类型的指针变量
1. 定义一个返回值为类指针的静态成员函数 （静态成员函数 可以通过类名调用，不需要先实例化对象通过对象调用）

```c
#include <iostream>
using std::cout;
using std::endl;

// 单例模式
// 该对象不能是一个栈对象, 全局/静态对象，只能是堆对象

class Singleton {
   public: 
    // 3. static 因为需要在类外访问，且在没有对象时通过此函数创建唯一对象，因此只能是静态函数，可以在类外通过 类名:: 直接调用
    static Singleton* getInstance() {
    //2. 创建对象指针，只有第一次调用getInstance时 才创建对象
    // 之后所有的调用，都直接返回第一次创建的对象指针，保证了唯一
        if (nullptr == _pInstance) {
            _pInstance = new Singleton();
        }
        return _pInstance;
    }

    static void destroy() {
        if (_pInstance) {
            //在类内部执行delete表达式
            delete _pInstance;
        }
    }

   private:
    // 1. 将构造函数和析构函数私有化,只能在类内部调用
    Singleton() { cout << "Singleton()" << endl; }
    ~Singleton() { cout << "~Singleton()" << endl; }

    // 2. 对象指针 保存唯一对象，
    // 4. static只能访问static数据成员
    static Singleton* _pInstance;
    
    // 此时还没有定义任何数据成员，单例模式模型
};
// static数据成员 只能在类外 初始化
Singleton* Singleton::_pInstance = nullptr;

int main(void) {

    // new表达式在执行的过程中会创建对象,一定会调用构造函数
    // Singleton * ps1 = new Singleton;//error
    // Singleton * ps2 = new Singleton;//在类之外无法操作

    Singleton* ps1 = Singleton::getInstance();
    Singleton* ps2 = Singleton::getInstance();
    cout << "ps1:" << ps1 << endl;
    cout << "ps2:" << ps2 << endl;

    // delete ps1;//error 可以完成功能，但是不优雅,将析构函数 static，调用
    Singleton::destroy();
    
    return 0;
}
```

- 读取配置文件（程序的输入   方便修改配置，不需要直接改源码）

## 动手实现一个循环队列

```c
#include <iostream>
using std::cout;
using std::endl;

class Queue {
   public:
    Queue(int capacity = 10)
        : _capacity(capacity),
          _front(0),
          _rear(0),
          _base(new int[_capacity]()) {
        cout << "Queue(int)" << endl;
    }

    ~Queue() {
        if (_base) {
            delete[] _base;
            _base = nullptr;
        }
        cout << "~Queue()" << endl;
    }

    bool empty() const;
    bool full() const;
    void push(int number);
    void pop();
    int front() const;
    int back() const;

   private:
    int _capacity;
    int _front;
    int _rear;
    int* _base;
};

bool Queue::empty() const { return _front == _rear; }

bool Queue::full() const { return (_rear + 1) % _capacity == _front; }

void Queue::push(int number) {
    if (!full()) {
        _base[_rear++] = number;
        _rear %= _capacity;
    } else {
        cout << "queue is full, cannnot push any data" << endl;
    }
}

void Queue::pop() {
    if (!empty()) {
        _front++;
        _front %= _capacity;
    } else {
        cout << "queue is empty, no more data!" << endl;
    }
}

int Queue::front() const { return _base[_front]; }

int Queue::back() const { return _base[(_rear - 1 + _capacity) % _capacity]; }

void test0() {
    Queue que;
    cout << "que is empty?" << que.empty() << endl;
    que.push(1);
    cout << "que is empty?" << que.empty() << endl;
    for (int idx = 2; idx < 12; ++idx) {
        que.push(idx);
    }
    cout << "que is full?" << que.full() << endl;

    cout << "pop an element" << endl;
    que.pop();
    cout << "que's front element:" << que.front() << endl;
    cout << "push an element" << endl;
    que.push(10);
    cout << "que's back element:" << que.back() << endl;
    cout << "que is full?" << que.full() << endl;

    while (!que.empty()) {
        cout << que.front() << " ";
        que.pop();
    }

    cout << "\nque is empty?" << que.empty() << endl;
}

int main(void) {
    test0();
    return 0;
}
```

### 流的四种状态：

- `badbit`表示发生系统级的错误，如不可恢复的读写错误。通常情况下一旦`badbit`被置位，流就无法 再使用了。
- `failbit`表示发生可恢复的错误，如期望读取一个数值，却读出一个字符等错误。这种问题通常是可 以修改的，流还可以继续使用。
- 当到达文件的结束位置时，`eofbit`和 `failbit`都会被置位。
- `goodbit`被置位表示流未发生错误。如果`badbit`、`failbit`和`eofbit`任何一个被置位，则检查流状态的 条件会失。


### 缓冲区

缓冲区又称为缓存，它是内存空间的一部分。也就是说，在内存空间中预留了一定的存储空间，这些存 储空间用来缓冲输入或输出的数据，这部分预留的空间就叫做缓冲区。缓冲区根据其对应的是输入设备 还是输出设备，分为输入缓冲区和输出缓冲区。

1. 全缓冲区： 只有当缓冲区满的时候，才会执行刷新操作
1. 行缓冲区：碰到换行符的时候，进行刷新
1. 非缓冲区：不带缓冲区，有多少数据就刷新多少

使用操纵符

- `endl`: 用来完成换行，并刷新缓冲区
- `~~ends~~`~~: 在输入后加上一个空字符，但是**没有刷新缓冲区**（这个需要注意，很多书上说可以刷新缓冲区）~~
- `flush`: 用来直接刷新缓冲区的
- `~~unitbuf~~`~~: 在每次执行完写操作后都刷新输出缓冲区~~
- `~~nounitbuf~~`~~: 让流回到正常的缓冲方式~~

`cout`

- cout 一般情况下，没有遇到换行符，不会刷新缓冲区
- cout遇到换行，立刻刷新缓冲区，此时，可看作 行缓冲
- cout缓冲区默认1024个字节的空间 （ 此时全缓冲），也会立刻刷新缓冲区

#### 日志系统的设计

1.  服务器程序的运行特点：7 * 24小时 不间断运行<br />**守护进程（面试常考点**）： 没有终端进行交互<br />记录其运行轨迹 --> 日志系统的工作 
1.  日志系统 是**服务器**程序**必须要提供的模块** 
1.  日志类型： 
   - 系统日志
   - 业务型日志

<a name="log4cpp"></a>
#### log4cpp

-  Category  日志来源 
-  Layout      日志布局 
-  Priority     日志优先级 
-  Appender( 终端、文件)  日志目的地 

`Category::shutdown( );`  // 释放申请空间，回收日志系统

1个`Category`可以对应多个`Appender`

- OstreamAppender
- FileAppender
- RollingFileAppender  回卷文件

1个`Appender`要对应1个`Layout`

- BasicLayout
- PatternLayout

不同的Appender设置Layout时，不能使用相同的Layout对象,必须要再**建立一个新的Layout对象

<a name="BasicLayout"></a>
#### BasicLayout

```c
#include <iostream>
#include <log4cpp/BasicLayout.hh>
#include <log4cpp/Category.hh>
#include <log4cpp/OstreamAppender.hh>
#include <log4cpp/Priority.hh>

using std::cout;
using std::endl;
using namespace log4cpp;

int main(void) {
    // 1. 创建目的地对象
    OstreamAppender* posAp = new OstreamAppender("console", &cout);
    // 2. 设置目的地的日志布局
    posAp->setLayout(new BasicLayout());

    // 3. 创建Category对象
    Category& root = Category::getRoot();
    // 4. 设置好优先级
    root.setPriority(Priority::DEBUG);  // 给Category对象设置优先级

    root.setAppender(posAp);

    // 5. 进行日志记录
    root.emerg("this is an emerge message");  //本条日志的级别是emerg级别
    root.alert("this is an alert message");
    root.crit("this is a crit message");
    root.error("this is an error message");
    root.warn("this is a warn message");
    root.notice("this is a notice message");
    root.info("this is an info message");
    root.debug("this is a debug message");

    // 6. 释放申请资源,回收日志系统
    Category::shutdown();

    return 0;
}
```

<a name="4b7bde9f"></a>
#### pattern layout

-  －X: X信息输出时左对齐 
-  %p: 输出日志信息优先级，即DEBUG，INFO，WARN，ERROR，FATAL, 
-  %d: 输出日志时间点的日期或时间，默认格式为ISO8601，也可以在其后指定格式，比如：%d{yyyy-MM-dd HH:mm:ss,SSS}，输出类似：2011-10-18 22:10:28,921 
-  %r: 输出自应用启动到输出该log信息耗费的毫秒数 
-  %c: 输出日志信息所属的类目，通常就是所在类的全名 
-  %t: 输出产生该日志事件的线程名 
-  %l: 输出日志事件的发生位置，相当于%C.%M(%F:%L)的组合,包括类目名、发生的线程，以及在代码中的行数。 
-  %x: 输出和当前线程相关联的NDC(嵌套诊断环境),尤其用到像java servlets这样的多客户多线程的应用中。 
-  %%: 输出一个"%"字符 
-  %F: 输出日志消息产生时所在的文件名称 
-  %L: 输出代码中的行号 
-  %m: 输出代码中指定的消息,产生的日志具体信息 
-  %n: 输出一个回车换行符，Windows平台为"\r\n"，Unix平台为"\n"输出日志信息换行 

```c
#include <iostream>
#include <log4cpp/Category.hh>
#include <log4cpp/OstreamAppender.hh>
#include <log4cpp/PatternLayout.hh>
#include <log4cpp/Priority.hh>

using std::cout;
using std::endl;
using namespace log4cpp;

int main(void) {
    // 0. 创建PatternLayout对象
    PatternLayout* ptnLayout = new PatternLayout();
    ptnLayout->setConversionPattern("%d [%c] [%p] %m%n");

    // 1. 创建目的地对象
    OstreamAppender* posAp = new OstreamAppender("console", &cout);
    // 2. 设置目的地的日志布局
    posAp->setLayout(ptnLayout);

    // 3. 创建Category对象
    Category& root = Category::getRoot();
    // 4. 设置好优先级
    root.setPriority(Priority::DEBUG);  // 给Category对象设置优先级

    root.setAppender(posAp);
    Category& module1 = root.getInstance("register");

    // 5. 进行日志记录
    module1.emerg("this is an emerge message");  //本条日志的级别是emerg级别
    module1.alert("this is an alert message");
    module1.crit("this is a crit message");
    //本条日志的级别小于Category的优先级级别，就会被过滤掉
    module1.error("this is an error message");
    module1.warn("this is a warn message");
    module1.notice("this is a notice message");
    module1.info("this is an info message");
    module1.debug("this is a debug message");
    module1.debug("xxxxxx");

    // 6. 释放申请资源,回收日志系统
    Category::shutdown();

    return 0;
}
```

日志可以同时往 多个对象 写入

一直使用FileAppender往文件中写入，会有什么问题？

占用磁盘空间会越来越大，比如视频存储的场景，不希望日志系统占用过多的空间，因此需要 回卷文件。

对于日志系统，只能1G的空间，希望每一个日志文件大小不要超过100M，一共会产生10个日志文件。

当1G的空间快用完时，可以将**最早产生的日志数据删除**，如何**再写新的**日志数据。

<a name="b45978b6"></a>
#### RollingFileAppender  // 回卷文件

```c
#include <iostream>

#include <log4cpp/Category.hh>
#include <log4cpp/Priority.hh>
#include <log4cpp/PatternLayout.hh>
#include <log4cpp/OstreamAppender.hh>
#include <log4cpp/FileAppender.hh>
#include <log4cpp/RollingFileAppender.hh>

using std::cout;
using std::endl;
using namespace log4cpp;


int main(void)
{
    // 0. 创建PatternLayout对象
    PatternLayout* ptnLayout = new PatternLayout();
    ptnLayout->setConversionPattern("%d [%c] [%p] %m%n");

    PatternLayout* ptnLayout2 = new PatternLayout();
    ptnLayout2->setConversionPattern("%d [%c] [%p] %m%n");

    PatternLayout* ptnLayout3 = new PatternLayout();
    ptnLayout3->setConversionPattern("%d [%c] [%p] %m%n");

    // 1. 创建目的地对象
    OstreamAppender* posAp = new OstreamAppender("console", &cout);
    // 2. 设置目的地的日志布局
    posAp->setLayout(ptnLayout);

    FileAppender* fileApp = new FileAppender("fileApp", "wd.log");
    //不同的Appender设置布局时，不能使用相同的Layout对象
    //必须要再建立一个新的
    fileApp->setLayout(ptnLayout2);

    auto rollingFileApp =
        new RollingFileAppender("rollingFileApp", "rollingwd.log", 5000, 9);
    rollingFileApp->setLayout(ptnLayout3);

    // 3. 创建Category对象
    Category& root = Category::getRoot();
    // 4. 设置好优先级
    root.setPriority(Priority::DEBUG);  // 给Category对象设置优先级
    root.setAppender(posAp);
    root.addAppender(fileApp);
    root.addAppender(rollingFileApp);
    Category& module1 = root.getInstance("register");

    for (int idx = 0; idx < 100; ++idx) {
        // 5. 进行日志记录
        module1.emerg("this is an emerge message");  //本条日志的级别是emerg级别
        module1.alert("this is an alert message");
        module1.crit("this is a crit message");
        //本条日志的级别小于Category的优先级级别，就会被过滤掉
        module1.error("this is an error message");
        module1.warn("this is a warn message");
        module1.notice("this is a notice message");
        module1.info("this is an info message");
        module1.debug("this is a debug message");
        module1.debug("xxxxxx");
    }

    // 6. 释放申请资源,回收日志系统
    Category::shutdown();

    return 0;
}
```
